D3h01
Platform for sharing and flexing code screenshots / snippets where users can like and comment on posts. Users have to post code snippets that they wrote and liked and other users can see who all have posted their snippets. The snippets should be in text form(can be copied) and not screenshots/photos with syntax highlighting.
  • Task 1 - Share snippets option and page for each snippet
  • Task 2 - Give reactions and suggest edits to the shared snippets
D3h02
Develop a solution for hostel management. The solution should provide features for allocation of room to the, Reallocation of room, no of rooms empty, no of students from a particular institute in a particular hostel building etc. The application should allow report generation (student wise, building wise, no of rooms empty /occupied etc.) The solution should assume more than one hostel building, no. of floors in each building. No. of rooms on each floor. Requirements on each floor.
  • Task 1 - Online leave application system with status updates for the application (rejected or approved)
  • Task 2 - Complaint raising system for students in hostels related to water, hygiene, electrical appliances.
D3h03
All in one app where we can have complete smart services for university related to cleanliness, lost & found, canteen orders, event notifications, etc. You think standing in the queue for ordering food and then waiting for the food to get ready is wasting time? Yeah me too. How if we could just order the food after (after the lectures/before reaching the canteen) itself and the food gets ready before you get there. So, you’ve found an earphone and not sure where you’re supposed to report it? Found dirt in your classroom and don’t know the appropriate authority to report? Could we just directly report this in the app or web app and the appropriate authority will take care of the situation.
  • Task 1 - Vehicle/ any other tools sharing system inside the campus, where people can list things to lend or others can openly request for required items.
  • Task 2 - Voting system for any event and decisions related to mess, venue and dates of events
D3h04
Bill split - develop an application in which users can track their expenses and split the bills among their friends.
  • Task 1 - Graph for monthly expenditure
  • Task 2 - Notification to each person after split
D3h05
Develop a solution for mess management. The solution should provide facilities to the mess admin and the students. For mess admins, they can manage coupons and other necessary details like menu, and pricing in the mess dashboard. Students can buy coupons from the mess dashboard only, deciding their desired meals for the week. The aim of the project will be to remove the hassle of buying coupons and provide a centralized platform for mess management.
  • Task 1 - QR codes for each meal a day in place of paper coupons
  • Task 2 - Razorpay Integration
D3h06
PM2.5 Prediction
  • Project Description: The project is mainly based on supervised learning part of Machine Learning i.e. Regressions and Classifications. The working of the models is based on the various algorithms from the sklearn library. The major focus would be to achieve optimum accuracy for the Machine Learning model and deploying of the model using any desired framework (eg. Flask).
  • Problem Statement: On the given dataset, perform Exploratory Data Analysis (i.e., Nan values, finding correlations, visualizing, finding insights, outlier detection, etc.) and Feature Engineering (Performing cleaning/processing of Data, feature selection, outlier removal, etc.). Then create appropriate models to predict pm2.5 and evaluate the model using various evaluation techniques. The project submission should be a deployed and hosted model on your website.
  • View Dataset
  • Task 1 - Perform Cross validation
  • Task 2 - Hyper parameter tuning, documentation
D3h07
Web Automation for purchasing tickets on irctc using Selenium. Handle Tatkal ticket purchases, etc.
  • Task 1 - Giving inputs for web drivers with help of apis (eg. - fastApi, flask, Django, Node
  • Task 2 - Automate the food ordering process in IRCTC for passengers
D3h08 *
Create a simple crowdfunding app in which creators can put up their ideas and the crowd can fund them with crypto currencies/ethers.
  • Task 1 - The website should at least be capable of handling metamask wallets.
  • Task 2 - If storing any additional data, the usage of any decentralized storage like ipfs or pinata wil be a plus (not mandatory)
D3h09 *
Create an on-chain event ticketing system where event organizers can create their show/event and attendees can purchase entry tickets with crypto currencies/ethers.
  • Task 1 - The website should at least be capable of handling metamask wallets.
  • Task 2 - If storing any additional data, the usage of any decentralized storage like ipfs or pinata wil be a plus (not mandatory)
* Note
  • All the smart contracts should be deployed and verified on any working test network. For example, if working with ethereum(preferred), the working test networks are Goerli, Sepolia
  • You are free to use any development environment for testing and deploying of your contract. Eg truffle, hardhat
  • Every project should have a frontend which would interact with the smart contract using any integration library such as ethers js, web3js or Moralis etc.
